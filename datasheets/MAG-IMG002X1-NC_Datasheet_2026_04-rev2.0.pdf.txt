3. Description  
The MAG -IMG00 2X1-NC is a radiation -hardened F ull 
HD CMOS digital image sensor  with an active -pixel array 
of 1920Hx1080V . It can come in both a monochrome  
(MAG -IMG00 241-NC) as in a color version (MAG -
IMG00 251-NC). Both versions implement rolling 
shutter readout strategy . 
The image sensor outputs raw sensor image data 
through a parallel CMOS output interface (3v3) , which 
is configurable to output 10 or 12 bit pixel  data. 
Synchronization signals such as the pixel clock (P IXCLK), 
data valid (DVAL), line valid (LVAL) and frame valid 
(FVAL) are also sent  along through the same output 
interface , which can run up to 100 MHz . The imager can 
be configured to capture data in trigger mode, in which 
it can capture one image or multiple images (burst 
mode), or in continuous mode (video mode). It also 
includes features such as image mirroring/reversing  
and sub -sampling /decimation  to ease the integration 
for the end application . 
All the above configurations can be programmed 
through a SPI interface which can run up to 20 MHz.

5. Block diagram  
The block diagram of the imager’s internal s is depicted in Figure 3. The core of the device is the pixel array, together 
with the peripherals to drive and read the pixels.  The latter circuitry includes the column load, sample and hold and the 
column ADC. SRAM is implemented to store the data to be sent out through the parallel output bus as raw image data.  
 
Figure 3: Imager block diagram  
 
The signals that are required to select the right column and row to be read , are generated by the sequencer in the 
digital. This block also generates the signals that are used to drive the reset and selection signals of the pixels , and the 
signals that are use d for the sample and hold and the ADC. The sequencer defines  the biggest part of the digital circuitry 
on chip. Next to the sequencer, the digital contains an SPI interface together with a register map which is accessible 
through this SPI interface. It allows the user to configure the device in the operating mode wanted , including the 
implemented features . The digital  operates at the digital supply voltage VDDD of 1.8V.  
Next to the digital, the imager contains analog circuitry to generate on -chip reference currents and voltages, and 
reference clocks as well. The chip contains a PLL which generates the clock for the digital, as well as the clocks required 
to drive the ADCs , the output MUX and the output interface. The PLL requires an external clock reference of 20 Mhz.

8. Functional  description  
The MAG -IMG00 2X1-NC is a radiation -hardened Full HD CMOS digital image sensor with an active -pixel array of 
1920Hx1080V. It can come in both a monochrome (MAG -IMG00 241-NC) as in a color version (MAG -IMG00 251-NC). 
Both versions implement rolling shutter readout strategy . 
The image sensor outputs raw sensor image data through a parallel CMOS output interface (3v3), which is configurable 
to output 10 or 12 bit pixel data. Synchronization signals such as the pixel clock (P IXCLK), data valid (DVAL), line valid 
(LVAL) and frame valid (FVAL) are also sent along through the same output interface, which can run up to 100 MHz  
maximally . The imager can be configured to capture data in trigger mode, in which it can capture one image or multiple 
images (burst mode), or in continuous mo de (video mode). It also includes features such as  windowing,  image mirroring  
and sub -sampling to ease the integration at the customer.  
All the above configurations can be programmed through a SPI interface which can run up to 20 MHz.  
8.1 Sensor architecture  
8.1.1 Pixel Array Structure  
Figure 5 shows the pixel array  of the image sensor. The pixel core of the imager  contains the pixel array along with its 
drivers and encoders. The pixel array is 1920x1080  pixels  as active area  (= Full HD) , and in  addition there are  64 rows of 
black pixels to calibrate the black color (3 2 at the top and 3 2 at the bottom). These pixels are optically black  as they are 
shielded by on -chip metal . Additional transparent dummy pixels  (8 rows  and 8 columns)  are added to improve image 
uniformity within the active area. A guard ring/barrier concludes the pixel array.  
 
 
Figure 5: Pixel Array core  
 
8.1.2 Monochrome vs. color  
To improve the efficiency of the pixel, there are micro -lenses ( MLA) deposited on each pixel, centered around the 
photo diode. These micro -lenses are present on both the monochrome  as on the color  version. The difference between 
the monochrome version and the color version is the addition of a color filter array (CFA) in the case of the color sensor. 
This CFA is there to filter the spectrum for each pixel  in order to be able to create a color image. The pattern that is used 
is the well -known Bayer pattern, in particular the BayerRG (RGGB). The array is pictured in Figure 6. 
8.1.3 Default Readout Order  
By convention, the sensor core pixel array is shown with the first addressable (logical) pixel (0,0) in the left bottom 
corner (see Figure 6). In the case of the color version (MAG -IMG00 251-NC), red is the first pixel at (0,0), as the used

8.2.4 Operating modes  
The device can be in three different modes  of operation, defined by  ‘AcquisitionCommand_Value ’: 
• STOP mode: AcquisitionCommand_Value  = 1 
• CONTINUOUS mode: AcquisitionCommand_Value = 8 
• MULTIFRAME mode: AcquisitionCommand_Value = 9 
8.2.4.1  STOP mode  
To enter this mode, ‘AcquisitionCommand_Value ’ needs to be set to 1.  
The imager is not grabbing  any images in this state and can be considered as  an idle state. Note that for most 
configuration settings (except exposure times settings), it is mandatory to return to this mode before making changes.   
8.2.4.2  CONTINUOUS mode  
To enter this mode, ‘AcquisitionCommand_Value ’ needs to be set to 8.  
This is a “video” mode that will continue to grab frames as long as ‘AcquisitionCommand_Value ’ is set in this state. It 
will stop doing this when STOP  (=1) is written into the register.  This mode is the default start -up mode after booting the 
imager. So, it will start sending out images after correct initialization . Before you make any changes to the imager 
configuration, it is mandatory to go to the STOP mode first.  
8.2.4.3  MULTIFRAME mode  
To enter this mode, ‘AcquisitionCommand_Value ’ needs to be set to 9.  
A limited number of frames will be grabbed when ‘AcquisitionCommand_Value ’ is set to MULTIFRAME (9). The imager 
will reset the ‘AcquisitionCommand_Value ’ register back itself to STOP (1) when this sequence has been grabbed. The 
number  of frames that will be grabbed can be defined in the ‘AcquisitionFrameCount_Value ’ register.  
While t he STOP and CONTINUOUS mode are self -explanatory and do not require additional information , the 
MULTIFRAME is however a bit more complex as it can be triggered in different ways , i.e. an auto trigger mode and a 
triggered trigger mode . 
• Trigger mode  auto  (AUTO_TRIG) : TriggerMode_ Valu e = 0 
 
In this trigger mode the register  ‘TriggerMode_Value ’ is set to  0, which means that when ‘AcquisitionCommand_Value ’ 
is set to MULTIFRAME mode (setting 9) , it will automatically start grabbing a number of images as defined in 
‘AcquisitionFrameCount_Value ’. After grabbing these images, the imager will return to the STOP state. A new frame 
capture can be started by putting the device again in the MULTIFRAME mode . 
 
• Trigger mode trigger  (WAIT_TRIG) : TriggerMode_Value  = 1 
 
In this trigger mode the register  ‘TriggerMode_Value ’ is set to  1, which means that when ‘AcquisitionCommand_Value ’ 
is set to MULTIFRAME mode (setting 9) it will not automatically start grabbing a number of images, but it will await an 
external trigger . This external trigger can be given by pin trigger (TRIG_IN pin) , or by a software trigger (register trigger) , 
as should be defined through the ‘TriggerSource_Value ’ register bit . Also note that the number of frames to be captured 
is given by the register ‘TriggerAdditionalParameter_Value ’, which is different from the AUTO_TRIG mode.

8.4 SPI interface  and programming  
A Serial Peripheral  interface (SPI) is provided via which MAG -IMG00 2X1 can be fully configured using the SPI protocol , 
at a maximum clock frequency of 20MHz  (typical is 10 MHz) . The SPI interface is configured as a slave and t he interface 
uses clock polarity 0 and clock phase 0, as shown in  Figure 12. Both master and slave update data on the falling clock 
edge, while they both read data on the rising edge. The reset and chip select signals are both asynchronous active low 
signals. A SPI command is 64 bits in total and consists of the targeted memory address ( 31-bit), a read (‘1’) or write (‘0’) 
selection bit, and the actual  32-bit data transfer , as shown in Figure 12. 
Programming the image sensor is done by writing the appropriate values to the on -board configuration registers. In 
section 12, more information can be found related to the  (configuration)  registers . 
Note that the MISO pin of the SPI slave interface does not have a tri -state mode! This means that the driver is always 
on. As a result this SPI slave interface cannot be used in parallel with other SPI slave interfaces.

9.4 Recommended operating conditions in radiation environment  
The main performance parameter that is impacted by the irradiation is the dark current. The result of the increased 
dark current, is that the image can saturate faster for long exposure times. In terms of functionality there is in principle 
no difference. The imager is still fully functional. Due to on -chip parameter drifts, some biasing parameters might require 
some re -tuning during its lifetime. In practice, this is a simple upload of a register setting through the CoaXPress link 
and does not require manu al maintenance on the camera itself. Note that after booting, the camera anyways needs a 
register map upload with the correct configuration settings.  
 
Additionally, the d ark current  also increases with increased temperature, and this is the case for all CMOS image sensors. 
The sensor’s self  heating is  however also  dependent on the dose rate it receives. So a higher dose rate means a higher 
temperature increase, which leads to higher dark current.  
 
For optimal operation of the sensor, the following advises  are proposed  when operating the sensor at high doses and 
high dose rates (+1kGy/hr):  
• Keep exposure time below 5ms, and preferably as low as possible.  
o The higher the exposure time, the more dark current is integrated. See the videos for our tested 
conditions.  
o Note that the exposure time is easily programmable through uplink CXP commands. So it can be 
changed on the fly!  
• Keep the temperature low:  
o This is a system’s engineering solution, where you could actively cool the sensor, so it will not heat up. 
In this case the above advise do not necessarily have to be followed.  
• When the dose rate is lower (e.g. <0.5kGy/hr), the sensor temperature will not increase much, so you don’t 
suffer much from the heating.

11.2 Handling  
11.2.1  ESD and handling  
Imaging sensors must be handled in an ESD safe area. A ground strap is required when handling the sensors in a non -
ESD safe area.  
A few guidelines when handling image sensors:  
• Please w ear mouth protection ( a face mask) to minimize the risk of glass lid contaminati on through saliva or 
other particles.  
• Please w ear clean gloves that are ESD safe. Contaminated or dirty gloves need to be changed or cleaned.  Finger 
tips of the gloves should be tight to reduce the risk of contaminating the glass lid.  
• Make sure to always handle the image sensor at the package, and avoid touching the glass lid. When delivered, 
a Kapton tape is protecting the glass.  
• Avoid touching the glass lid in all cases.  Finger grease can etch optical coatings and cause permanent damage.  
• Only remove the Kapton tape from the glass at the very last stage.  
 
11.2.2  Cover glass cleaning  
If by any reason the cover glass is contaminated, it must be cleaned  in an ESD -safe environment , preferably also wearing 
a wrist -band . Gloves and mouth protection  should be worn at all times .  
Loose particles on top of the cover glass can be blown off using clean nitrogen or clean ionized dry air.  
Wet cleaning can be done with IPA (Isopropylalcohol) on a clean wipe.  Wipe gently over the complete cover glass and 
avoid using the same part of the wipe again . Use a small intermediate dispenser for the IPA to avoid contamination of 
the IPA in the bottle.  Avoid saturating the wipe as using too much IPA  may leave drying stains afterwards.  
Select a suitable wipe. Be aware that some ESD protective wipes easily scratch optical coatings on the cover glass.  
Suitable examples are Berkshire Super Polx 1200 and Berkshire Durx 670.  
It is always preferred to keep the sensors clean and avoid the need for cleaning.  Custom ers are advised to evaluate their 
cover glass cleaning process before implementing  it on production material . 
 
11.2.3  Storing  
If stored for longer time , the devices need to be stored  in a moisture -proof, vacuum -sealed and anti -static bag . It should 
be stored in a temperature and humidity -controlled environment. The storage temperature should not be higher than 
50°C, while the humidity should be below 60%.
