Heading 1: Document purpose and scope
This document is part of PSU00001 product and defines the test strategy, test setup and the detailed test procedures to perform electrical characterisation and production test on the product. 
All the information in this document is intended to describe the verification tests that will be done on the product mentioned above. Any major deviations from this plan will be discussed prior or at latest during the test readiness review (TRR) meeting. Major changes will be listed in the revision history. Major changes can be (list is not exclusive):
Number of DUT’s 
Test conditions
Hardware/equipment changes
Test facility changes
Addition or removal of tests
Any update to a test procedure that changes the test coverage or testing strategy 
Minor changes are kept in the revision history but not explicitly informed to the rest of the team. These changes are limited to:
Small changes in a test procedure, e.g. additional SPI commands, increasing wait times
Correction of typo’s 
Any changes (major and minor) after the TRR meeting will be communicated via email towards the team and if needed discussed during a meeting. 

Heading 1: Test strategy 
This chapter describes the test methods used for the verification and validation of the MAG-PSU00001 product. The test strategy aims at maximal test coverage of the product.
Functional testing and electrical parametric testing of the test structures is used to verify the performance. 

Heading 2: Test conditions
During functional verification and characterization testing the temperature and supply voltage of the DUT’s will be varied to validate the ASIC specifications over the defined operating conditions. The conditions listed in this section are the minimal set of conditions that will be tested. Heading 3: Connectivity test (CONN)
The connectivity test is the initial stage of any other test. It checks for shorts on the DUT interface. DUTs need to successfully pass this test-stage in order to be tested further. The CONN tests are designed to reduce the risk of destructive errors damaging the DUT or setup.

Heading 3: Characterization (CHAR)
The characterization stage is intended to test the electrical requirements of the DUTs over the operating conditions (temperature, load-current and supply-level) as defined in section 2.1.  
During the verification phase of the product development lifecycle the number of parts used in the analysis can be smaller compared to the number used in validation phase. The full characterization as part of the device validation phase should be performed on a number of parts that is statistically meaningful. The test coverage should be similar in both phases. If not, the reason for changing test coverage must be addressed in the characterization report. 

Heading 3: Characterization of the OTP block (CHAR_OTP)
Special test stage where only the behaviour of the OTP block is characterized, since this block reacts on temperature changes, a temperature sweep is needed and this is not compatible with the CHAR test stage since test time will explode. 

Heading 3: Final Test (FT)
The final test stage is intended to verify that the chip is within specifications. This is a reduced test compared to the characterization (CHAR). Where the CHAR test does a high resolution sweep over parameters, the FT only checks if the minima and maxima are within spec. 

Heading 3: DCDC-Module testing (MODULE)
The DCDC-Module test stage is performed on MAG-PSU10XX1 assemblies. Here the efficiency and voltage-ripple are important. Tests like line-regulation and load-regulation are performed as well. When these parameters are within test limits, the module is considered good.

Heading 3: Vision DCDC sub-board testing (VIS)
The Vision sub-board test stage is performed on the DCDC sub-board of the MAG-VIS400xx PCBs,  in order to verify the functionality of the board. An expected load-profile, mimicking that of the other boards in the stack, can be applied. If the voltage-level and ripple are within test limits, the board is considered functional verified.

Heading 3: Calibration (CAL)
The calibration step contains a couple test procedures that are performed on an empty socket or empty base-board configuration. They are meant to be used to characterize the setup itself. This includes the measurement equipment but also the fixed pull-up or pull-down resistors in the setup. This will allow for more accurate analysis of the actual Device Under Test (DUT) in the other test stages.

Heading 3: Overview
Figure 3 provides an overview of the complete test setup used for most tests outlined in the test procedures (Chapter 5). As shown in the diagram, the Devices Under Test (DUTs) are placed within a temperature-controlled environment, which may be maintained using a thermostreamer to ensure stable and controlled test conditions. The DUTs are connected to various measurement instruments via a connection matrix, allowing flexible signal routing.
To enable automated test execution, all instruments are connected through Ethernet and centrally controlled by a test PC. This setup ensures efficient data acquisition, measurement synchronization, and automated execution of the defined test procedures.

Table 7: Socket board probe-points

Heading 3: Measurement board
The measurement board will serve as an interface between the daughter board and the power supplies, microcontroller, and other required equipment. These connections will be made using a combination of banana-connectors and SMA connectors. Additionally, the measurement board will function as a load board.
The design will support connections for a single daughter board (single-site testing). To ensure compatibility with different test environments, the measurement board will be designed for use with a thermostreamer.
Each measurement board will be assigned a unique serial number for identification. 
All supply domains will be individually accessible on this board.

Heading 3: Power supplies 
Power is supplied to the DUT using bench Power Supply Units (PSUs). The setup contains two power domains. One main power-supply (VDD), that powers the DUT, and the other a 3.3V power-domain for the microcontroller and integrated mux.
The power-up and power-down sequence for default-connected supplies is detailed in Section 3.2.3. To prevent voltage overshoot, it is strongly recommended to use power sources with a controllable soft ramp.

During  testing, the initial state is different from the park state. This Power-down state is a more conservative state. Heading 3: Power cycle routines
Power-up sequence:
Disconnect all measurement equipment
Apply PSU-3v3 = PSU-supply = 0V
Wait 10ms
Ramp up PSU-3v3 to its defined operating condition 3.3V in 10ms or slower
Wait 10ms
Set IO’s as defined in the park state routine (section 3.2.2)
Ramp up PSU-supply to its defined operating condition 5V-11V in 10ms or slower
Power-down sequence:
Ramp down PSU-supply to 0V in 10ms or slower
Ramp down PSU-3v3 to 0V in 10ms or slower
Disconnect all measurement equipment
