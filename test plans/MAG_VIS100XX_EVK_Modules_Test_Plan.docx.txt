Heading 1: Document purpose and scope
This document is part of MAG-VIS Modules project and defines the test strategy, test setup and the detailed test procedures to perform electrical and functional verification. Radiation testing is out of scope of this test plan, these test are not done on individual modules.
All the information in this document is intended to describe the verification tests that will be done on the product mentioned above. Any major deviations from this plan will be discussed prior or at latest during the test readiness review (TRR) meeting with the customer. Major changes will be listed in the revision history. Major changes can be (list is not exclusive):
Number of DUT’s 
Test conditions
Hardware/equipment changes
Test facility changes
Addition or removal of tests
Any update to a test procedure that changes the test coverage or testing strategy 
Minor changes are kept in the revision history but not explicitly informed to the customer. These changes are limited to:
Small changes in a test procedure, e.g. additional SPI commands, increasing wait times
Correction of typo’s 
Any changes (major and minor) after the TRR meeting will be discussed with the customer.

This chapter describes the test methods used for the verification and validation of the MAGVIS modules. The test strategy aims at maximal test coverage of the product.

Heading 3: Visual inspection (VI)
The purpose of the external visual inspection test stage is to assess the physical condition of the Device Under Test (DUT) before testing. This stage helps identify any visible defects, such as cracks, issues with pins, delamination, or other mechanical damages that may have occurred during handling, or transportation. By conducting a thorough visual examination, potential issues can be detected early, preventing further testing on compromised components and ensuring the reliability of the test results.

Heading 3: Functional verification (VER)
The functional verification stage is intended to confirm that the product is compliance (functional and/or parametric) against its documentation for normal use-case(s).

Heading 2: Test hardware

Heading 3: Overview
Figure 2 provides an overview of the complete test setup used for most tests outlined in the test procedures (Chapter 4). As shown in the diagram, the Devices Under Test (DUTs) are placed within a temperature-controlled environment, which may be maintained using a thermostreamer or temperature chamber to ensure stable and controlled test conditions. 
To enable automated test execution, all instruments are connected through Ethernet and centrally controlled by a test PC. This setup ensures efficient data acquisition, measurement synchronization, and automated execution of the defined test procedures.
Figure 2: Measurement setup overview 
The MAG-VIS100xx modules require an adapted measurement configuration to facilitate the functionality of the MAG-DRV IC. The adapted setup incorporates the following additional equipment:
Oscilloscope: Required for waveform capture and timing analysis. This verifies the operation of the MAG-DRV IC, including successful generation of ON/OFF output signals and the generation of PWM signals.
Magics MUX Card: A multiplexer card is integrated to multiplex the outputs of the MAG-DRV IC to the oscilloscope.
Power resistors: These resistors are added to limit the current flow trough the drivers of MAG-DRV IC.
The measurement setup overview diagram for the MAG-VIS1000x will be constrained to highlight only the new, module-specific test equipment. The core system components (Frame Grabber, PSU, and Control Software) are mandatory elements of the EVK test platform and will be intentionally omitted from the specific diagram to maintain clarity and avoid unnecessary clutter.

Heading 3: DUT information
To limit the use of equipment channels, the DUTs are measured using a time-multiplexed approach, ensuring that only one device is tested at a time (single-site test setup). 
Table 7 provides a detailed overview of all DUT pins and specifies the corresponding test equipment required for each pin to execute the defined test procedures. 
For debugging purposes, additional connections to the test equipment may be beneficial. These optional connections are marked with the postfix "∗" in the table. The following codes are used in the matrix:
HI_F: Forced voltage or current at the pin
HI_S, LO_S: Sensed voltage or current at the pin
LO_F: Return path
HI_IN: Sensed voltage or current at the pin connects to HI input of DMM or scope
LO_IN: Sensed voltage or current at the pin connects to LO input of DMM 
CHx: Input of oscilloscope with x in range [1 to 4]

Heading 3: Pin load requirements
During the hardware development attention needs to be given to following items: 
Low-ohmic connections 
For ground and power connections.
Impedance matched signals
TRXP/TRXN: The TRXN pin is connected to a Coax transmission line leading to the frame grabber, while the TRXP pin is terminated on the substrate board. Matching must adhere to the 75Ω transmission line standard. 

Heading 3: Power supplies 
Power is supplied to the DUT using bench Power Supply Units (PSUs). All DUT supply channels are listed in Table 8, where the "default connected" column indicates whether a channel is connected by default during testing unless otherwise specified in the test procedure. The supply can either be directly connected to the pin, via a relays or via an DMMI for most accurate supply current measurements. The PSU channels can be split over different bench equipment depending on availability and sourcing/measurement requirements. 
The power-up and power-down sequence for default-connected supplies is detailed in Section 3.2.3. To prevent voltage overshoot, it is strongly recommended to use power sources with a controllable soft ramp.
