Heading 1: Document purpose and scope
This document is part of the MAG-IMG002X1 project and defines the test strategy, test setup and the detailed test procedures to perform electrical verification, functional verification and radiation testing on the product.
This document also provides a detailed test schedule and lists test facilities. 
All the information in this document is intended to describe the verification tests that will be done on the product mentioned above. Any major deviations from this plan will be discussed prior or at latest during the test readiness review (TRR) meeting with the customer. Major changes will be listed in the revision history. Major changes can be (list is not exclusive):
Number of DUT’s 
Test conditions
Hardware/equipment changes
Test facility changes
Addition or removal of tests
Any update to a test procedure that changes the test coverage or testing strategy 
Minor changes are kept in the revision history but not explicitly informed to the customer. These changes are limited to:
Small changes in a test procedure, e.g. additional SPI commands, increasing wait times
Correction of typo’s 
Any changes (major and minor) after the TRR meeting will be discussed with the customer. 

Heading 1: Test strategy 
This chapter describes the test methods used for the verification and validation of the MAG-IMG002X1 product. The test strategy aims at maximal test coverage of the product.
Functional testing and electrical parametric testing of the test structures is used to verify the performance. Radiation tests will be performed to evaluate hardness against ionizing radiation.  

Heading 2: Test conditions
During functional verification and characterization testing the temperature and supply voltage of the DUT’s will be varied to validate the ASIC specifications over the defined operating conditions. The conditions listed in this section are the minimal set of conditions that will be tested. If additional test conditions are needed to validate certain parameters, then they are explicitly mentioned in the test procedure. 
Temperature conditions are mentioned in this document with following codes:
RT = room temperature = 25°C +/- 5°C

Heading 3: Visual inspection (VI)
Visual inspection is an examination of the external surfaces, construction, marking, and workmanship of a finished packaged device. Since the lid of the product is of glass, bond pad corrosion, wire bonding and glass lid inspection is included. This stage helps identify any visible defects, such as cracks, issues with pins, delamination, or other mechanical damages that may have occurred during manufacturing, handling, or transportation. By conducting a thorough visual examination, potential issues can be detected early, preventing further testing on compromised components and ensuring the reliability of the test results. 
Visual inspection is done on parts where the glass lid (Kapton) protector is removed. This test stage must be performed in a cleanroom or under a vertical laminar flow hood. JESD22 test method B101 is used as reference for the visual inspection. 

Heading 3: Final testing (FT)
The final test stage is intended to confirm that the product is compliance (functional and/or parametric) against the test limits that are defined for each of the test parameters in the test plan. The goal is to screen devices for manufacturing defects and remove mavericks.
For flight models (FM) or production models (PROD) this test stage will be used for most of the electrical testing of the product. During this test stage the glass lid is protected by the Kapton tape and the light input is blocked towards the sensor. 
In this setup configuration, the temperature of the DUT can be precisely controlled, enabling reliable acquisition of dark images. Optical test parameters like dark current and dark signal non-uniformity (DSNU) are measured during this test stage.

Heading 3: Optical testing (OT)
During optical testing, a light source provides input to the image sensor, and images are captured at different light intensity levels. Key optical parameters - such as responsivity, photo-response non-uniformity (PRNU), and temporal noise - are measured, and a pixel defect map is generated. Product differentiation (binning) is then performed based on the number of detected defect pixels, clusters, and column or row defects.
Optical testing is done on parts where the glass lid (Kapton) protector is removed. This test stage must be performed in a cleanroom or under a vertical laminar flow hood.

Heading 3: Camera PCB functional test (VIS)
This test stage is intended to perform a functional verification of the IMG002x1 product after assembling on the PCB VIS4000x. This PCB is used in the camera module. During this test stage test patterns from the IMG002x1 are captured via the parallel interface and compared to the expected value.  

Heading 3: Functional verification (VER)
The functional verification stage is intended to confirm that the product is compliance (functional and/or parametric) against its documentation for normal use-case(s). This stage usually uses a selection of tests and test conditions from the characterization test stage.

Heading 3: Electrical Characterization (CHAR)
The characterization stage is intended to test the electrical requirements of the DUTs over the operating conditions (temperature and supply) as defined in section 2.1.  
During the verification phase of the product development lifecycle the number of parts used in the analysis can be smaller compared to the number used in validation phase. The full characterization as part of the device validation phase should be performed on a number of parts that is statistically meaningful. The test coverage should be similar in both phases. If not, the rationally for changing test coverage must be addressed in the characterization report. 

Heading 3: Optical characterization (CHAR_OT)
During optical characterization, a light source provides input to the image sensor, and images are captured at different light intensity levels. Key optical parameters - such as responsivity, photo-response non-uniformity (PRNU), and temporal noise - are measured, and a pixel defect map is generated.
Optical testing is done on parts where the glass lid (Kapton) protector is removed. This test stage must be performed in a cleanroom or under a vertical laminar flow hood.

Heading 3: Trimming (TRIM)
The trimming stage is intended obtain the best-fit trim codes for the building blocks. Since there is no on-chip non-volatile memory available, trimming is done per wafer lot on a set of at least 20 devices. Median values for each trim code is calculated in postprocessing and stored in a new revision of the camera configuration files.   

Heading 3: TID radiation testing (TID)
The Total Ionizing Dose (TID) test stage evaluates the effects of cumulative radiation exposure on the Devices Under Test (DUTs). Dose rates are chosen to be close to target applications of the product. 

Heading 3: Annealing (ANNEAL)
The annealing test stage at room temperature is conducted to observe any recovery effects in the Devices Under Test (DUTs) following radiation exposure. After the TID test, the DUTs are left at similar biasing conditions at ambient temperature for 24 hours to allow for potential charge redistribution and partial recovery of radiation-induced degradation. This stage helps assess the transient and permanent effects of radiation damage, providing insights into the behaviour of the DUTs.

Heading 3: Aging (AGING)
The aging test stage at 100°C for 168 hours evaluates the long-term stability of the DUTs. By subjecting the devices to an extended period of elevated temperature, this test simulates accelerated aging conditions, helping to identify potential degradation mechanisms such as drift in electrical parameters, leakage currents, or shifts in threshold voltage. This stage is particularly useful for assessing how radiation-induced defects evolve over time when exposed to heat. DUTs are placed in a controlled temperature chamber, with their biasing conditions based on prior test stages. 

Heading 3: Light power calibration (CAL)
The light power generated by the light source in the optical test setup requires to be calibrated on a periodical base. This information is required for the test parameters that are calculated back to the number of photons, like the responsivity of the imager. This test stage requires a dedicated setup where a interface board connects the photodiode to the measurement board.

Low-ohmic connections

Shielded/guarded signals
All connections between the function generator, scope, SMU and the pins of the ASIC should be done with coax or triax cables. Pay attention to the shielding of these signals.  
Impedance matched signals
Make sure the SPI lines are length matched as well as the video signals. The video signals should also have an impedance of 50 Ohm.

Heading 3: Daughter board and external components 
A dedicated daughter board will be designed to serve as an interface between the DUT and the measurement board. Make sure the SPI lines are length matched as well as the video signals. The video signals should also have an impedance. The impedance of the video lines on the daughterboard should not be impedance matched at 50 Ohm because a Ron measurement on these pin would be disturbed by this impedance matching. The impedance match will be foreseen on the measurement board with relays and 50 Ohm resistors. No testpoints are added on the socket PCB.
Each daughter board will have a unique serial number for identification.

Heading 3: Measurement board
The measurement board will serve as an interface between the daughter board and the connection matrix, power supplies, FPGA, and other required equipment. These connections will be made using a combination of flat cables, SMA connectors, and power connectors. Additionally, the measurement board will function as a load board.
The design will support connections for a single daughter board (single-site testing). To ensure compatibility with different test environments, the measurement board be designed for use with a thermostreamer.
Each measurement board will be assigned a unique serial number for identification. 
All supply domains will be individually accessible on this board.
The measurement board will contain relays and muxes to perform onboard muxing. Every video signal/SPI pin as well as the reset and trigger pin can be connected to either the communication board (CL/CXP) or to the measurement equipment. For the videolines, a path to the communication board (CL/CXP) can be made via relays and through 50Ohm resistors to have impedance matched lines. A 20 MHz clock will be generated onboard as well. For the test busses, a unity gain buffer can be used if needed for dynamic signals. An RC filter circuit will be foreseen on the measurement board on the reset pin as well. Debug connectors will be foreseen for the video lines and SPI lines. The relays and muxes will be steered by a microcontroller that communicates, via a different SPI bus than the imager uses, with the communication board (CL/CXP) in order to set the relays and muxes in the correct way. The 20 MHz clock, microcontroller, relays and muxes will be powered by another supply in order be able to monitor the current consumption of the imager correctly. For both the analog and digital test busses, a possibility is created to use an extra external buffer for voltages. While testing with these buffers, these buffers showed clipping outputs to supply rails when the common mode input signal exceeds 2.8V. For this reason, the decision was made to not use these buffers while measuring analog signals.
